## Technical Architecture Highlights

### AI Recognition Pipeline
1. User snaps photo → Upload to S3
2. Call Google Cloud Vision API (<2 sec)
3. Return foods with confidence scores
4. User confirms → Save to database

**AI Options:**
- **Recommended:** Google Cloud Vision ($1.50/1K images, 90%+ accuracy, <2 sec)
- **Future:** Custom TensorFlow model (92%+ accuracy, lower per-unit cost at scale)

### Data Privacy & Security
- Photos encrypted at rest (S3 AES-256)
- Presigned URLs (7-day expiration)
- HTTPS for all data in transit
- GDPR/CCPA compliant (export, delete, portability)
- Supplements always private (no sharing option)

### Database Schema
- `food_logs` table (user_id, photo_url, privacy_level, shared_with, nutrition totals)
- `food_items` table (individual foods within a meal)
- `water_logs` table (hydration tracking)
- `supplement_logs` table (always private, encrypted)
- Simple RLS policies (no stored procedures, follows FitCircle architecture rules)

## Key Risks & Mitigations

### Risk 1: AI Accuracy Below Expectations (High Risk)
- **Mitigation:** Set expectations ("AI may not recognize all foods"), show confidence scores, easy editing
- **Contingency:** If <80% accuracy, prioritize manual entry UX

### Risk 2: Low Free → Pro Conversion (Medium Risk)
- **Mitigation:** Clear value prop, 7-day free trial, limit free tier (3 photos/day)
- **Contingency:** If <15%, gate feature entirely to Pro

### Risk 3: Privacy Concerns (Medium Risk)
- **Mitigation:** Private by default, granular sharing controls, no data selling
- **Contingency:** Add "Privacy First" badge, emphasize privacy in marketing

### Risk 4: Low Engagement / Habit Formation (High Risk)
- **Mitigation:** Gamification (XP, streaks), push notifications, social accountability, <30 sec logging
- **Contingency:** If D7 <25%, increase streak bonuses, daily challenges
